# mDocks vs Obsidian

Obsidian is a powerful Markdown-based knowledge base. mDocks is a focused Markdown reader for local files, folders, and GitHub repo docs. They overlap in file format, but they solve different problems.

## Choose Obsidian when

Obsidian is a strong fit if you want:

- Backlinks and graph views
- Daily notes
- Personal knowledge management
- Plugins and custom workflows
- A long-term writing and note-taking environment

## Choose mDocks when

mDocks is a better fit if you want:

- A simple reader for Markdown folders
- A browser-based way to open local Markdown files
- GitHub repository documentation browsing
- A focused reading interface with a file tree and table of contents
- Local-first reading without building a second brain system

## Different center of gravity

Obsidian is centered on creating and connecting notes. mDocks is centered on reading technical documentation that already exists.

That makes mDocks useful for project docs, repo docs, runbooks, implementation plans, and Markdown folders where the main job is reading rather than knowledge management.

## Can they work together?

Yes. You can write or organize Markdown in your preferred editor, including Obsidian, and use mDocks as a clean reading surface for selected folders or repository docs.
